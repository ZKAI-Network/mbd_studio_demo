# MBD Studio App – LLM Reference

## Introduction

This app is a **feed builder studio** for creating personalized content feeds. It has three main product modes:

- **Build**: Search → Stories → Features → Scoring → Ranking → SDK
- **Deploy**: Feed Algos → Feed Configs → Warmup
- **Eval**: Dashboard

**Main flow (Build)**:
1. **Search**: Choose index (farcaster-items, farcaster-users, zora-coins, polymarket-items, polymarket-wallets, token-items, wallet-users, kalshi-items, kalshi-wallets-v20260315), run query (filter/sort, boost, semantic, AI prompt, ES query), get results.
2. **Stories**: Generate stories (e.g. polymarket_v2).
3. **Features**: Enrich hits with user-item features (user from farcaster-profiles, zora-profiles, polymarket-wallets).
4. **Scoring**: Rerank with models (e.g. polymarket-rerank-v1).
5. **Ranking**: Sort/diversity/limits on features/scores.
6. **SDK**: Code editor runs mbd-studio-sdk; code is generated from last calls.

**Key components**:
- `App.js`: Product switcher (build/deploy/eval), step navigation.
- `GlobalContext`: API key, base URL, account, currentContext (for help agent).
- `BuildContext`: selectedIndex, search/stories/features/scoring/ranking APIs, hits, lastCall objects.
- `DeployContext`: deployApi (algos, configs).
- `EvalContext`: evalSteps.
- `CardRouter`: Maps hit._index to FarcasterItemCard, ZoraCoinCard, PolymarketItemCard, PolymarketWalletCard, KalshiItemCard.

## Source Code (Compact, Layout + Event Logic)

### App.js
```javascript
const AppProviders = ({ children }) => (
  <GlobalProvider>
    <BuildProvider>
      <DeployProvider>
        <EvalProvider>{children}</EvalProvider>
      </DeployProvider>
    </BuildProvider>
  </GlobalProvider>
);

const BuildContent = ({ activeStep }) => {
  switch (activeStep) {
    case 0: return <SearchService />;
    case 1: return <StoriesService />;
    case 2: return <FeaturesService />;
    case 3: return <ScoringService />;
    case 4: return <RankingService />;
    case 5: return <FeedBuilder />;
    default: return null;
  }
};

const DeployContent = ({ activeStep }) => {
  switch (activeStep) {
    case 0: return <FeedsService />;
    case 1: return <FeedConfigsService />;
    case 2: return <WarmupService />;
    default: return null;
  }
};

const EvalContent = ({ activeStep }) => {
  if (activeStep === 0) return <DashboardPage />;
  return null;
};

const AppContent = () => {
  const { activeProduct, setActiveProduct } = useProduct();
  const { buildSteps } = useBuild();
  const { deploySteps } = useDeploy();
  const { evalSteps } = useEval();
  const [activeStep, setActiveStepState] = useState(0);
  const steps = activeProduct === 'build' ? buildSteps : activeProduct === 'deploy' ? deploySteps : evalSteps;

  // Tip: step navigation is guarded by `step.enabled`
  const setActiveStep = useCallback((step) => {
    const stepConfig = steps[step];
    if (stepConfig?.enabled) setActiveStepState(step);
  }, [steps]);

  // Switching product resets the flow to step 0
  const handleProductChange = useCallback((product) => {
    setActiveProduct(product);
    setActiveStepState(0);
  }, [setActiveProduct]);

  const renderContent = () => {
    if (activeProduct === 'build') return <BuildContent activeStep={activeStep} />;
    if (activeProduct === 'deploy') return <DeployContent activeStep={activeStep} />;
    if (activeProduct === 'eval') return <EvalContent activeStep={activeStep} />;
    return null;
  };

  // Layout: AppBar + active step content
  return (
    <>
      <AppBar
        activeProduct={activeProduct}
        setActiveProduct={handleProductChange}
        activeStep={activeStep}
        setActiveStep={setActiveStep}
        steps={steps}
      />
      {renderContent()}
    </>
  );
};

function App() {
  return (
    <NeynarContextProvider
      settings={{
        clientId: "9af4d2c3-218f-4ce6-8cc8-5024d0065e28",
        defaultTheme: Theme.Light,
        eventsCallbacks: {
          onAuthSuccess: () => {},
          onSignout: () => {},
          onError: (error) => {},
        },
      }}
    >
      <AppProviders><AppContent /></AppProviders>
    </NeynarContextProvider>
  );
}
export default App;
```

## Tips for LLMs

1. **Index selection**: `selectedIndex` in BuildContext drives QueryColumn routing and filter options. Use `findIndex(hit._index)` when index may have a version suffix.
2. **Filter payload shape**: Filters use `{ filter, field, value, label, boost? }`. Filter types: term, terms, numeric, date, match, geo, is_null, not_null, custom, group_boost, terms_lookup, console_account.
3. **Search endpoints**: `/search/filter_and_sort`, `/search/boost`, `/search/semantic`, `/search/es_query`, `/search/ai_prompt`. Payload includes `index`, `origin: 'frontend'`, `feed_type`, `include_vector`.
4. **Hits enrichment**: Features and scores are merged into `hit._features` and `hit._scores`. Ranking writes `hit._ranking_score`. `getAvailableData(hits)` returns available feature and score keys.
5. **SDK code generation**: `code_generator.js` turns lastCall objects into mbd-studio-sdk chains. User context comes from features or scoring payload.
6. **currentContext**: Used by HelpPanel; includes product, page, source, queryBeingBuilt, numCandidates, etc.
